GET방식 요청 및 응답

        GET방식 : 요청하는 주소 뒤에 전달하려는 값을 key=value 형태로 전달하는 방식
        /ServletProject1/example1.do?inputName=김길동&inputAge=30
        / 프로젝트 이름 / 서비스 요청 주소 ?key=value&key=value&.....
                                            -> 전달하려는 값이 작성된 문자열 
                                               : QueryString

        - 장점 : 단순한 표현, 주소창에 값을 임의 작성 가능
                 캐싱(기억) 가능 -> 요청 주소가 저장됨 == 북마크, 즐겨찾기, 링크 공유 가능

        - 단점 : 보안에 취약 (패스워드 공유가능 등)
                 글자 수 제한이 존재

        * 언제 사용? 검색, 게시글, 뉴스 공유 등을 이용할 때

POST방식 요청 및 응답

        POST방식 : (GET방식과 반대) 주소 뒤에 전달하는 값이 붙는게 아닌
            HTTP Protocol의 Body부분에 숨겨져서 전달되는 방식

        - 장점 : 길이 제한 없음, 보안성 향상(데이터 직접 노출X)
                 
        - 단점 : 캐싱 불가능, 추가적 문자 인코딩 처리 필요
